## 💡 Was ich gelernt habe

- **Touched-Event**: Erkennt wenn ein Objekt berührt wird → `part.Touched:Connect(funktion)`
- **Parameter**: Die berührende Part wird automatisch als Parameter übergeben (`objectTouched`)
- **script.Parent**: Gibt das Objekt zurück, dem das Script gehört
- **Hierarchy**: `objectTouched.Parent` geht eine Ebene nach oben (z.B. Fuß → Charakter)
- **FindFirstChild("Humanoid")**: Prüft ob ein Charakter ein Spieler ist (gibt `nil` zurück wenn nicht)
- **FindFirstChildWhichIsA("Humanoid")**: Robustere Variante — sucht nach Klassen-Typ statt Name
- **Humanoid-Properties**: `humanoid.Health = 0` tötet Spieler, `humanoid.WalkSpeed` ändert Geschwindigkeit
- **Debounce-Pattern**: Boolean-Variable verhindert dass Events zu schnell mehrfach feuern
- **CanTouch**: Property um Touch-Erkennung an einem Part komplett ein/auszuschalten
- **task.wait(n)**: Pausiert Code für n Sekunden ohne alles zu blockieren
- **while-Loop**: `while bedingung do ... end` — läuft solange Bedingung true ist
- **String-Konkatenation**: `"Text" .. variable` (wie `+` in C#)
- **elseif-Ketten**: Für Medaillen-System (Gold/Silber/Bronze/Retry)

## 🔗 Vergleich zu C#

| Luau | C# |
|---|---|
| `objectTouched.Parent` | `gameObject.transform.parent` |
| `FindFirstChild("X")` | `GetComponent("X")` |
| `FindFirstChildWhichIsA("X")` | `GetComponent<X>()` |
| `task.wait(1)` | `await Task.Delay(1000)` |
| `"Hello" .. name` | `"Hello" + name` |
| `while x == true do` | `while (x == true) {}` |
| `humanoid.Health = 0` | `player.Health = 0` |
